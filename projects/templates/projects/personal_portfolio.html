This project is the website you are currently on!

Since most of my career I've worked on pre-existing projects, I started this to gain full experience from the beginning to the end of a project. I also started this as a way to learn more about Web Development, Source Control, Continuous Integration, and Cloud Platforms in general. This website was not created using any template or generation, it was all coded by me. Both front and back end, starting with nothing.

My current plan is to use this to showcase the experience I have gained throughout my career, as well as other personal projects I work on. However, I have no strict plans to keep it that way and may post other random stuff here as well. The end goal of this project is to have a place that I can grow and display my coding skills with what ever interesting ideas I come up with.

The code for this project, as well as all my other coding projects, can be found on my GitHub. I want to share and document my work here so that I am able to easily reference past work or to serve as an example for anyone looking to create something similar.

When starting this website, the only thing I knew was that I wanted to use Python in creating it. Simply because I enjoy using Python and at the time had not used it for much more than basic scripting. So naturally, my research led me to the Django Python Web Framework. Not only was Django Python based, but it used a Model-View-Template (MVT) design pattern. This interested me because web applications I worked on in the past at Fenway Group used a similar, although slightly different, Model-View-Controller (MVC) design pattern like Apache Struts and ASP.NET. Both of which were very old and out-dated frameworks. So it was nice to use a modern framework that popular sites like Instagram, Spotify, and Pinterest use.

In an MVT framework, like in an MVC framework, the Model manages the data, database interactions, and business logic. The View is not the same as the view in MVC however, and it is actually more like the Controller. The View is responsible for receiving user input and responding with the correct response. Much of what was done with Controllers in MVC is done already in the MVT framework itself. Lastly, the Template is responsible for rendering the data passed from the View in the form of an HTML web page.

After making an initial and very basic webpage, I realized the website was pretty ugly. So, I researched what CSS frameworks that other big websites using Django used and Bootstrap came up often. So I chose that, and it is what has allowed me to make the UI more bearable.

Once it looked pretty enough, I wanted to have it hosted on the web and came across Heroku, which is an affordable Cloud Platform as a Service (PaaS) owned by Salesforce with a lot of support specifically for Python and Django. Essentially, Heroku hosts and manages UNIX containers that are running gunicorn HTTP servers for Python web apps like mine. Heroku refers to these containers as dynos. Heroku also has very convenient Continuous Integration (CI) set up with GitHub so that I can easily deploy my application to the server after committing changes.

As I grew the website, I had an increasing amount of media like images and videos I wanted to post to the website. So I also set up WhiteNoise which is a Python library that simplifies serving these "static files". This should optimize the loading of these files through the use of compression and caching.

After all of that, I bought a domain name on SquareSpace Domains so that I can have a custom URL to share with people. Heroku allows me to add my custom domain to my web app and supplies me with a DNS target. This DNS target is needed to configure my domain using SquareSpace so that my web app is displayed when visiting www.pdani.online.
